# Veloserve Context

Veloserve is a hosting/control-panel product with existing work across:
- Web Server
- Web Panel
- Website
- Billing
- Infrastructure automation

The AI crew should continue development, improve quality, identify product gaps, create roadmap items, open GitHub issues, and prepare pull requests.

## Hard Rules

The AI crew must never:
- Deploy to production without human approval.
- Change billing/pricing without human approval.
- Email customers without human approval.
- Delete production data.
- Force push.
- Commit secrets.
- Modify production infrastructure directly.

Allowed automatically:
- Read repositories.
- Analyze code.
- Create reports.
- Create GitHub issues.
- Create local branches.
- Open pull requests.
- Run tests.
- Update documentation.

Human approval required:
- Merge PRs.
- Deploy production.
- Change pricing.
- Send customer-facing communication.
- Spend money.
- Access production servers.
