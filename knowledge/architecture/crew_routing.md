# Crew Routing Guide

## Route To `product_orchestrator_crew`

- cross-repo planning
- roadmap creation
- issue decomposition
- dependency mapping between products
- deciding which repo should own a feature

## Route To `veloserve_server_crew`

- server runtime
- HTTP handling
- PHP integration
- caching and performance
- config and virtual host behavior
- WordPress or Magento compatibility at the engine level

## Route To `velopanel_crew`

- panel backend
- panel UI
- account quotas
- domain and database workflows
- SSL, backups, monitoring, account/admin permissions

## Route To `platform_crew`

- `platform.veloserve.io`
- auth flows
- subscriptions and billing
- invoices, downloads, tickets
- dashboard UX and customer lifecycle

## Route To `licensing_crew`

- activation logic
- entitlement checks
- heartbeat flows
- license validation APIs
- product to license integration boundaries

## Route To `community_crew`

- community site IA
- engagement workflows
- public documentation discovery
- showcases and contribution flows

## Route To `shared_qa_security_crew`

- auth review
- ownership and permission checks
- secret handling
- billing and pricing risk review
- regression analysis
- production safety review
